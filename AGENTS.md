# Agents

This project uses the Payload CMS skill at `.agents/skills/payload/`.
Start with `.agents/skills/payload/SKILL.md` for a quick reference, then see `.agents/skills/payload/reference/` for detailed docs.

## Stack

- **Payload 3.84** on **Next.js 16** (App Router), React 19, TypeScript 5.7, TailwindCSS 4
- **Database**: SQLite via `@payloadcms/db-sqlite` (local file `Hypnotherapy.db`, configured by `DATABASE_URL=file:./Hypnotherapy.db` in `.env`)
- **Package manager**: pnpm (v9+ or v10). Uses `legacy-peer-deps=true` (`.npmrc`)
- **UI**: shadcn/ui (config in `components.json`), Radix primitives, Lucide icons

## Commands

| Task | Command |
|---|---|
| Dev server | `pnpm dev` |
| Lint | `pnpm lint` (eslint, extends `next/core-web-vitals` + `next/typescript`) |
| Lint fix | `pnpm lint:fix` |
| Build | `pnpm build` (also runs `next-sitemap` via `postbuild`) |
| Production | `pnpm build && pnpm start` |
| Integration tests | `pnpm test:int` (vitest, `tests/int/**/*.int.spec.ts`) |
| E2E tests | `pnpm test:e2e` (playwright, needs dev server at localhost:3000) |
| All tests | `pnpm test` (runs int then e2e) |
| Generate Payload types | `pnpm generate:types` |
| Generate import map | `pnpm generate:importmap` |
| Payload CLI | `pnpm payload <subcommand>` |

## Verification order

`lint` → `generate:types` (if schema changed) → `test:int` → `test:e2e`

## Architecture

```
src/
  payload.config.ts          # Entry point: collections, plugins, DB adapter
  payload-types.ts           # AUTO-GENERATED — do not edit manually
  app/
    (frontend)/              # Public website (Next.js App Router)
    (payload)/               # Admin panel — AUTO-GENERATED by Payload, do not edit
  collections/               # Pages, Posts, Media, Categories, Users
  blocks/                    # Layout builder blocks (Hero, Content, CTA, Archive, etc.)
  Header/ + Footer/          # Global configs + components
  plugins/index.ts           # Plugin wiring (redirects, nested-docs, SEO, form-builder, search)
  hooks/                     # Revalidation hooks
  access/                    # Access control functions
  fields/                    # Reusable field configs (defaultLexical, link, linkGroup)
  components/                # Shared React components
  utilities/                  # Helpers (ui/cn, getURL, mergeOpenGraph)
  search/                    # Search plugin field overrides + beforeSync
```

- Path alias `@/*` → `src/*` (tsconfig)
- Path alias `@payload-config` → `src/payload.config.ts` (used by generated admin code)
- `src/app/(payload)/layout.tsx` is **auto-generated** — never edit it
- `src/payload-types.ts` and `src/payload-generated-schema.ts` are **auto-generated** — run `pnpm generate:types` after schema changes

## Style & formatting

- Prettier: single quotes, no semicolons, trailing commas, printWidth 100 (`.prettierrc.json`)
- EditorConfig: 2-space indent, LF line endings
- shadcn component alias: `@/utilities/ui` (not `@/lib/utils`)
- TailwindCSS 4 — uses `@tailwindcss/postcss` plugin (not `tailwindcss` direct)

## Testing

- **Integration** (vitest): `tests/int/**/*.int.spec.ts` — jsdom env, uses `dotenv/config` setup, resolves `@/` paths via `vite-tsconfig-paths`
- **E2E** (Playwright): `tests/e2e/` — requires running dev server (`pnpm dev`), chromium only, seeds/cleans a test user via `tests/helpers/seedUser.ts`
- Test env overrides: `test.env` sets `NODE_OPTIONS="--no-deprecation --no-experimental-strip-types"`

## Database quirks

- SQLite file DB: `Hypnotherapy.db` in project root — schema pushes on dev (no migration step needed locally)
- No `pnpm payload migrate:create` / `pnpm payload migrate` needed for local dev (push mode is on)
- For production, switch to Postgres and set up migrations per Payload docs
- docker-compose.yml references **MongoDB** but the actual project uses **SQLite** — docker-compose is stale from template

## Env vars (`.env.example`)

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | SQLite file path or Postgres connection string |
| `PAYLOAD_SECRET` | JWT encryption key |
| `NEXT_PUBLIC_SERVER_URL` | Public URL, no trailing slash |
| `CRON_SECRET` | Authenticates scheduled job requests |
| `PREVIEW_SECRET` | Validates draft preview requests |
